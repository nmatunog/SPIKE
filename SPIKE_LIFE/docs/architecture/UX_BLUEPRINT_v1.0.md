SPIKE LIFE™
UX Blueprint 1.0
User Experience & Interaction Architecture
Version: 1.0
Purpose:
Define the user experience, navigation, interaction model, screen hierarchy, and design principles for SPIKE LIFE.
This blueprint ensures that every interface supports the educational objectives of SPIKE while remaining intuitive, engaging, and emotionally meaningful.

1. UX Vision
SPIKE LIFE is not a board game.
SPIKE LIFE is a Financial Life Simulator.
The interface should feel like:
* A modern fintech application
* A personal financial planner
* A life simulation experience
It should not feel like:
* Monopoly
* Excel
* A quiz app
* An insurance quoting tool

2. UX Principles
Every screen should satisfy four questions:
1. Where am I in life?
Current age
Life stage
Major milestones

2. How healthy am I financially?
Cash Flow
Protection
Goals
Retirement
Life Score

3. What just happened?
Life Event
Financial Impact
Personal Impact

4. What should I do next?
Top Recommendations
Player Decisions
Future Consequences

If any screen cannot answer these questions, redesign it.

10. Life Event Experience
Events interrupt the dashboard.
The player receives a focused modal.
Structure
Title
Narrative
Impact Summary
Decision Prompt

Example
"Congratulations!
You have been promoted."
Income increases.
Expenses may rise.
Would you like to:
• Save more
• Upgrade Lifestyle
• Fund Goals

Keep the emotional story before the numbers.

Support:

High-contrast mode

21. UX Guardrails
Every screen must:
Support a SPIKE learning objective.
Reduce cognitive load.
Prioritize decision-making over data entry.
Explain consequences without prescribing a single "correct" answer.
Use solution-oriented language rather than product-oriented language.
